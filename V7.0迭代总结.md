# V7.0 迭代总结 - 社区商业服务系统

## 迭代概述

本次 V7.0 迭代基于对系统全版本（V2.0～V6.0）的深度产品评审，发现并修复了**功能缺口**、**架构安全漏洞**和**数据一致性问题**，全面提升系统健壮性和用户体验。

---

## 一、产品评审发现的问题

### 评审方法
- 逐行审查三端后端代码（业务逻辑、接口安全、缓存策略）
- 检查所有 HTML 页面与对应 API 的覆盖情况
- 对比 V2.0～V6.0 迭代遗留的 "待办事项" 清单
- 验证测试用例的覆盖盲区

### 发现问题清单

| 优先级 | 类别 | 具体问题 |
|--------|------|----------|
| P0 | 数据库缺失 | 无 business_cart / business_order_items / business_notices 表，业务代码已写但表不存在会抛异常 |
| P0 | 架构安全 | 员工端(22312)未引入 rate_limiter，写接口无限流保护，存在刷接口风险 |
| P0 | 数据一致性 | 用户撤销申请/取消订单后，stats 缓存不清除，前端60秒内仍显示旧数据 |
| P1 | 功能缺口 | 管理端无公告(notices)管理页面，但公告API在用户端已投放 |
| P1 | 功能缺口 | 用户端无独立搜索页面，搜索历史API已实现但无入口 |
| P1 | 功能缺口 | 用户端无收藏夹页面，收藏API已实现但无界面入口 |
| P1 | 性能 | 员工端 statistics 接口每次都全表扫描，无缓存 |
| P2 | 测试覆盖 | 购物车流程、公告CRUD、订单明细、缓存一致性无测试用例 |
| P2 | 数据库 | business_products 无库存字段、business_venues/shops 无评分缓存 |

## 二、V7.0 修复与新增功能

### 2.1 数据库迁移（P0 修复）

**文件**：`business-common/migrate_v7.py`

| 变更项 | 类型 | 说明 |
|--------|------|------|
| business_cart | 新建 | 购物车表（user_id, product_id, quantity, selected，唯一约束防重复） |
| business_order_items | 新建 | 订单明细表（保存下单时价格快照，防止商品改价后订单金额变化） |
| business_notices | 新建 | 公告表（type/importance/status/start_date/end_date/软删除） |
| business_products | 增加字段 | stock（库存）、unit（单位）、sales_count（销售数） |
| business_orders | 增加字段 | items_snapshot（商品快照JSON）、delivery_address（配送地址） |
| business_venues | 增加字段 | avg_rating（评分缓存）、review_count（评价数缓存） |
| business_shops | 增加字段 | avg_rating、review_count、images（门店图片） |

### 2.2 架构安全修复（P0 修复）

**文件**：`business-staffH5/app.py`

- **引入 rate_limiter**：所有写操作接口添加限流保护
  - `POST /api/staff/applications/<id>/process` → 30次/分钟
  - `POST /api/staff/feedback/<id>/reply` → 10次/分钟  
  - `POST /api/staff/bookings/verify` → 30次/分钟
- **引入 cache_service**：statistics 接口增加 60 秒缓存，与用户端/管理端保持一致

### 2.3 数据一致性修复（P0 修复）

**文件**：`business-userH5/app.py`

- `cancel_application()` 撤销申请后立即 `cache_delete(user_stats_{uid})`
- `cancel_order()` 取消订单后立即清除 user_stats 缓存
- 保证前端 stats 接口在用户操作后下次请求即时获取最新数据

### 2.4 公告管理（P1 功能补齐）

**新增文件**：
- `business-admin/notices.html` - 管理端公告管理页面
- `business-admin/app.py` 新增公告 API：
  - `GET /api/admin/notices` - 公告列表（支持关键词/状态/类型筛选、分页）
  - `POST /api/admin/notices` - 创建公告（XSS防护 + 审计日志 + 限流）
  - `GET /api/admin/notices/<id>` - 公告详情
  - `PUT /api/admin/notices/<id>` - 更新公告（XSS防护 + 审计日志）
  - `DELETE /api/admin/notices/<id>` - 软删除公告
  - `GET /api/admin/orders/<id>/items` - 订单明细查询

**`business-admin/index.html`** - 导航栏新增「📢 公告管理」入口

### 2.5 搜索页面（P1 功能补齐）

**新增文件**：`business-userH5/search.html`

功能特点：
- 三端联合搜索：场地 + 门店 + 商品（并行请求，速度快）
- Tab切换：全部 / 场地 / 门店 / 商品
- 搜索历史：自动保存、可清空
- 热门搜索：基于搜索频率统计
- 结果卡片：含图片、价格、类型标签、直达链接
- 防抖 + 缓存（session级别）

### 2.6 收藏夹页面（P1 功能补齐）

**新增文件**：`business-userH5/favorites.html`

功能特点：
- 按类型筛选：全部 / 场地 / 门店 / 商品
- 管理模式：切换「管理」后可直接取消收藏
- 图片展示 + 价格展示
- 无限滚动加载更多

**`business-userH5/index.html`** - 菜单新增「🔍 搜索」「❤️ 我的收藏」入口

### 2.7 测试用例增强（P2 修复）

**文件**：`tests/test_api.py`

新增测试类 4 个，共 **+15 条测试用例**：

| 测试类 | 用例数 | 说明 |
|--------|--------|------|
| TestCartFlow | 6 | 购物车获取、加入购物车参数校验、全选、下单流程 |
| TestNoticesAPI | 3 | 公告列表(公开)、按类型筛选、无效ID查询 |
| TestAdminNotices | 4 | 管理员列表、创建参数校验(x2)、完整创建+删除流程 |
| TestOrderItems | 2 | 无效订单明细查询、已有订单明细查询 |
| TestCacheConsistency | 2 | stats刷新后一致性、员工端缓存有效性 |

**修复**：docstring 中 Windows 路径 `\Users` 导致的 Unicode 转义语法错误
