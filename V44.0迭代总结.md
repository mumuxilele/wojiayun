## 二、核心交付内容

### 2.2 新增申请类型

#### FR-BP-054 大厦门禁卡申请

**表单字段**:
- 申请人姓名、电话、部门、职位
- 门禁卡类型（员工卡/访客卡/临时卡）
- 权限范围（多选：主入口/办公楼层/停车场/健身房/餐厅）
- 生效日期、失效日期
- 申请原因、备注

**审批流程**: 一级审批（物业审批）

**系统联动**: 审批通过后自动下发门禁权限

### 2.3 Token验证服务

**用户端验证**:
```
验证地址: https://wj.wojiacloud.com/api/user/info
返回字段: user_id, user_name, user_phone, ec_id, project_id
```

**员工端验证**:
```
验证地址: https://gj.wojiacloud.com/api/staff/info
返回字段: staff_id, staff_name, staff_phone, ec_id, project_id, roles
```

**装饰器**:
- `@require_external_user` - 验证用户Token
- `@require_external_staff` - 验证员工Token

### 2.4 企微推送集成

**推送场景**:
1. 审批结果通知（通过/拒绝）
2. 待审批提醒（推送给审批人）
3. 到期提醒（候餐椅、货梯等）
4. 申请即将开始提醒

**推送方式**:
- Markdown格式消息
- 文本消息
- 卡片消息

### 2.5 系统联动接口

| 系统 | 功能 | 触发条件 |
|------|------|---------|
| **门禁系统** | 自动下发/撤销权限 | 门禁卡申请审批通过/失效 |
| **梯控系统** | 货梯预约/可用性检查 | 货梯使用申请审批通过 |
| **暖通系统** | 加时鲜风供应申请 | 暖通申请审批通过 |
| **监控系统** | 录像调取审计日志 | 监控申请审批通过 |

## 五、技术亮点

### 5.1 Token验证流程

```
用户请求 → 提取Token → 调用外部认证API → 获取用户信息 → 继续业务处理
    ↓
  失败 → 返回401错误
```

### 5.2 系统联动流程

```
审批通过 → 判断申请类型 → 调用对应系统API → 记录联动日志 → 返回结果
    ↓
  门禁卡 → grant_access_permission()
  货梯   → reserve_cargo_lift()
  暖通   → request_hvac_overtime()
  监控   → log_monitor_query()
```

### 5.3 企微推送流程

```
审批完成 → 构建消息体 → 调用企微Webhook → 返回推送结果
    ↓
  成功 → 记录日志
  失败 → 降级处理（不影响主流程）
```
