# V42.0 产品迭代报告

**版本**: V42.0  
**日期**: 2026-04-06  
**迭代周期**: V41.0 → V42.0  
**核心主题**: 商品规格体系完善 + 问答系统 + 用户体验增强  

---

## 一、迭代背景

### 1.1 产品评审结论

通过对现有系统的深入评审，V41.0版本发现以下核心缺陷：

| 缺陷类型 | 问题描述 | 影响程度 |
|---------|---------|---------|
| **规格体系缺失** | 商品多规格属性管理不完整，前端无完整规格选择联动 | 高 |
| **问答系统空白** | 用户无法对商品提问，商家无法回复 | 高 |
| **秒杀体验不足** | 缺乏实时倒计时组件，用户参与感弱 | 中 |
| **会员数字卡缺失** | 无电子会员卡展示页，影响会员归属感 | 中 |
| **积分商城前端缺失** | 后端已完成但用户端页面不存在 | 高 |

### 1.2 V42.0 迭代目标

1. **建立完整的商品多规格属性体系** - 支持SKU矩阵生成与管理
2. **构建商品问答系统** - 用户提问、商家回复、追问闭环
3. **完善秒杀体验** - 实时倒计时、抢购氛围
4. **增强会员权益感知** - 数字会员卡、权益展示
5. **补全积分商城用户端** - 积分兑换完整链路

## 二、新增功能

### 2.1 商品多规格属性体系

#### 2.1.1 后端服务 `product_spec_service.py`

```
business-common/product_spec_service.py (450+行)
```

**核心功能**：
- `get_spec_groups()` - 获取商品规格组及规格值
- `add_spec_group()` - 添加规格组（支持批量规格值）
- `generate_sku_matrix()` - 笛卡尔积生成SKU矩阵
- `create_or_update_sku()` - 按规格组合创建/更新SKU
- `_cartesian_product()` - 笛卡尔积计算

**特色能力**：
- 支持规格值附加价格（如 "128GB:+100"）
- 自动生成SKU矩阵，避免规格组合遗漏
- 按规格组合唯一索引，防止重复SKU

#### 2.1.2 API端点

| 端点 | 方法 | 说明 |
|-----|-----|------|
| `/api/user/products/<id>/specs` | GET | 获取规格组列表 |
| `/api/user/products/<id>/sku/matrix` | GET | 获取SKU矩阵 |
| `/api/user/products/<id>/sku/create` | POST | 创建SKU |
| `/api/user/skus/<id>` | PUT | 更新SKU价格/库存 |
| `/api/admin/products/<id>/specs` | POST | 管理端添加规格组 |

#### 2.1.3 前端增强 `product_detail.html`

- 规格属性分组UI（颜色、尺码、容量等）
- 规格选择联动（基于笛卡尔积矩阵）
- 无货规格自动禁用
- 价格/库存随规格实时更新

## 四、数据库迁移 `migrate_v42.py`

### 4.1 新增表

```sql
-- 规格组表
CREATE TABLE business_product_specs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    product_id INT NOT NULL,
    spec_name VARCHAR(50) NOT NULL,
    sort_order INT DEFAULT 0,
    deleted TINYINT DEFAULT 0,
    created_at DATETIME,
    updated_at DATETIME
);

-- 规格值表
CREATE TABLE business_product_spec_values (
    id INT AUTO_INCREMENT PRIMARY KEY,
    spec_id INT NOT NULL,
    spec_value VARCHAR(100) NOT NULL,
    extra_price DECIMAL(10,2) DEFAULT 0,
    sort_order INT DEFAULT 0,
    deleted TINYINT DEFAULT 0
);

-- 问教主表
CREATE TABLE business_product_qa (
    id INT AUTO_INCREMENT PRIMARY KEY,
    product_id INT NOT NULL,
    user_id INT NOT NULL,
    content TEXT NOT NULL,
    status ENUM('pending','answered','closed') DEFAULT 'pending',
    created_at DATETIME,
    updated_at DATETIME
);

-- 回复表
CREATE TABLE business_product_qa_replies (
    id INT AUTO_INCREMENT PRIMARY KEY,
    qa_id INT NOT NULL,
    user_id INT NOT NULL,
    reply_type ENUM('answer','followup') NOT NULL,
    content TEXT NOT NULL,
    created_at DATETIME
);

-- 追加评价表
CREATE TABLE business_order_append_reviews (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id INT NOT NULL,
    product_id INT NOT NULL,
    user_id INT NOT NULL,
    content TEXT NOT NULL,
    created_at DATETIME
);
```
