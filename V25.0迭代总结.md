## 二、核心变更

### 2.1 F1: 积分商品管理页面 (P0)

**问题**: V23.0新增的积分商城系统后端API已实现，但管理端缺少商品管理页面

**解决方案**: 新建完整的积分商品CRUD管理页面

**新增文件**: `business-admin/points-mall.html`

**核心功能**:
1. 积分商品列表展示（分页、筛选）
2. 添加/编辑积分商品（名称、分类、价格、库存、限购）
3. 上架/下架管理
4. 删除确认（防止误删）
5. 图片上传预览

**页面元素**:
| 字段 | 类型 | 说明 |
|------|------|------|
| goods_name | 文本 | 商品名称 |
| category | 选择 | 分类（实物/虚拟/优惠券） |
| points_price | 数字 | 积分价格 |
| original_price | 数字 | 原价（划线价） |
| total_stock | 数字 | 总库存 |
| per_limit | 数字 | 每人限购数量 |
| sort_order | 数字 | 排序权重 |
| goods_image | 图片 | 商品图片 |
| description | 多行文本 | 商品描述 |

### 2.3 F3: 运营数据看板页面 (P1)

**问题**: 管理端只有基础统计页面，缺少可视化数据分析能力

**解决方案**: 新建完整的运营数据看板页面

**新增文件**: `business-admin/data-board.html`

**核心功能**:
1. **收入趋势图** - 近30天收入折线图
2. **收入来源分布** - 饼图展示各来源占比
3. **转化漏斗图** - 访问→下单→支付→完成
4. **热销商品TOP10** - 横向条形图
5. **消费会员TOP10** - 会员消费排行榜
6. **数据导出** - 支持导出Excel

**ECharts图表配置**:
```javascript
// 收入趋势配置
trendChart.setOption({
    tooltip: { trigger: 'axis' },
    legend: { data: ['收入', '订单数'] },
    xAxis: { type: 'category', data: days },
    yAxis: [
        { type: 'value', name: '收入', axisLabel: { formatter: '¥{value}' } },
        { type: 'value', name: '订单' }
    ],
    series: [
        { name: '收入', type: 'line', smooth: true, data: incomeData },
        { name: '订单数', type: 'bar', data: orderData }
    ]
});

// 来源分布饼图
sourceChart.setOption({
    tooltip: { trigger: 'item', formatter: '{b}: ¥{c} ({d}%)' },
    series: [{
        type: 'pie',
        radius: ['40%', '70%'],
        data: sourceData,
        itemStyle: { borderRadius: 8 }
    }]
});
```

### 2.4 F4: 管理端API扩展

**问题**: 新增页面需要对应的管理端API支持

**解决方案**: 在 `business-admin/app.py` 中扩展API端点

**新增API端点**:

#### 积分商品管理
| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/admin/points-mall/goods` | 获取商品列表 |
| GET | `/api/admin/points-mall/goods/<id>` | 获取商品详情 |
| POST | `/api/admin/points-mall/goods` | 创建商品 |
| PUT | `/api/admin/points-mall/goods/<id>` | 更新商品 |
| DELETE | `/api/admin/points-mall/goods/<id>` | 删除商品 |
| PUT | `/api/admin/points-mall/goods/<id>/status` | 更新状态 |

#### 拼团活动管理
| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/admin/group-buy/activities` | 获取活动列表 |
| GET | `/api/admin/group-buy/activities/<id>` | 获取活动详情 |
| POST | `/api/admin/group-buy/activities` | 创建活动 |
| PUT | `/api/admin/group-buy/activities/<id>` | 更新活动 |
| DELETE | `/api/admin/group-buy/activities/<id>` | 删除活动 |
| POST | `/api/admin/group-buy/activities/<id>/start` | 启动活动 |
| POST | `/api/admin/group-buy/activities/<id>/pause` | 暂停活动 |
| GET | `/api/admin/group-buy/activities/<no>/orders` | 获取活动订单 |
