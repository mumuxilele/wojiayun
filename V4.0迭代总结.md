## 🔴 P0 严重问题修复（已完成）

### 1. 员工端订单详情缺少数据隔离
- **文件**: `business-staffH5/app.py`
- **函数**: `get_order_detail`
- **问题**: 员工查询订单详情时只检查了 `id`，没有验证 `ec_id/project_id`
- **影响**: 员工可查看不属于自己项目的订单详情
- **修复**: 添加 `ec_id/project_id` 过滤条件

### 2. 员工端订单状态更新缺少数据隔离
- **文件**: `business-staffH5/app.py`
- **函数**: `update_order_status`
- **问题**: 更新订单状态时没有验证订单归属
- **影响**: 员工可修改不属于自己项目的订单
- **修复**: 更新前验证订单是否属于当前员工管辖范围

### 3. 批量删除申请单缺少数据隔离
- **文件**: `business-admin/app.py`
- **函数**: `admin_batch_applications`
- **问题**: 批量操作只用了 IN 子句，没有添加数据范围过滤
- **影响**: 可能跨项目删除或修改申请单
- **修复**: 添加 `ec_id/project_id` 过滤条件，并记录审计日志

## 🟠 P1 重要功能增强（已完成）

### 1. 预约超时取消脚本审计日志
- **文件**: `business-common/cancel_expired_bookings.py`
- **问题**: 脚本只打印日志到控制台，没有写入数据库
- **修复**: 新增 `log_to_audit()` 函数，自动取消时写入审计日志

### 2. 新增收藏功能
- **新增文件**: `business-common/business_favorites.sql`
- **新增接口**:
  - `GET /api/user/favorites` - 获取我的收藏列表
  - `POST /api/user/favorites` - 添加收藏
  - `DELETE /api/user/favorites/<type>/<id>` - 取消收藏
  - `GET /api/user/favorites/check` - 检查是否已收藏
- **功能**: 用户可收藏场馆、门店、商品

### 3. 新增优惠券系统
- **新增文件**: 
  - `business-common/business_coupons.sql` - 优惠券表结构
  - `business-admin/coupons.html` - 优惠券管理页面
- **新增接口（管理端）**:
  - `GET /api/admin/coupons` - 优惠券列表
  - `POST /api/admin/coupons` - 创建优惠券
  - `PUT /api/admin/coupons/<id>` - 更新优惠券
  - `POST /api/admin/coupons/<id>/issue` - 发放优惠券
- **新增接口（用户端）**:
  - `GET /api/user/coupons` - 获取我的优惠券
  - `POST /api/user/coupons/exchange` - 积分兑换优惠券
  - `GET /api/coupons/available` - 获取可兑换的优惠券
- **优惠券类型**:
  - 现金券：直接抵扣金额
  - 折扣券：按比例折扣（如8折）
  - 积分券：赠送积分

### 4. 统计页面刷新优化
- **文件**: `business-admin/statistics.html`
- **问题**: 刷新按钮没有loading状态反馈
- **修复**: 刷新时显示loading状态，并行加载所有数据

## 🟡 P2 优化建议（待后续迭代）

- 有的用 `access_token` 参数，有的用 `Token` header
- 建议：统一使用 `access_token` 查询参数

### 2. 缺少API文档
- 建议：引入 Swagger/OpenAPI 文档

### 3. 缺少自动化测试
- 建议：编写单元测试和集成测试

### 4. 预约时段选择体验问题
- 当前只能选择连续时段
- 建议：支持不连续时段选择
