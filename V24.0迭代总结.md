## 二、核心变更

### 2.1 F1: 微信支付V3正式集成 (P0)

**问题**: payment_service.py 多处TODO，支付无法正式上线

**解决方案**:
- 微信统一下单API调用（Native支付）
- 回调AES-GCM解密验签
- 配置从环境变量读取

**修改文件**: `business-common/payment_service.py`

**环境变量**:
```
WECHAT_APP_ID
WECHAT_MCH_ID
WECHAT_SERIAL_NO
WECHAT_API_V3_KEY
WECHAT_PRIVATE_KEY
WECHAT_NOTIFY_URL
```

**新增方法**:
- `_create_wechat_pay_url()` - 统一下单获取code_url
- `_wechat_sign()` - RSA2签名
- `verify_wechat_callback()` - AES-GCM解密验签

### 2.4 F4: 拼团活动系统 (P1)

**问题**: 营销活动单一，缺少社交电商能力

**解决方案**: 新建完整的拼团服务模块

**核心功能**:
1. 拼团活动管理（创建/上下架）
2. 参与拼团（创建订单）
3. 成团判定（自动成功/失败退款）

**数据库表**:
- `business_group_activities` - 拼团活动表
- `business_group_orders` - 拼团订单表

**API设计**:
- 员工端: 活动CRUD
- 用户端: 参与拼团、查询我的拼团

### 新增表结构

**business_group_activities** (拼团活动表)
```sql
CREATE TABLE business_group_activities (
    id INT AUTO_INCREMENT PRIMARY KEY,
    activity_no VARCHAR(30) NOT NULL UNIQUE,
    name VARCHAR(100) NOT NULL,
    product_id INT NOT NULL,
    product_name VARCHAR(100),
    product_image VARCHAR(500),
    original_price DECIMAL(10,2),
    group_price DECIMAL(10,2) NOT NULL,
    min_people INT DEFAULT 2,
    max_people INT DEFAULT 100,
    current_people INT DEFAULT 0,
    valid_hours INT DEFAULT 24,
    status VARCHAR(20) DEFAULT 'pending',
    ...
```

## 五、系统架构变化

### 支付流程（修改后）
```
用户下单
  ↓
选择支付渠道 (微信/支付宝)
  ↓
调用统一下单API (真实请求)
  ↓
获取支付链接/二维码
  ↓
用户支付
  ↓
回调验签 + 解密
  ↓
更新订单状态 + 发放积分
```

### 物流查询（修改后）
```
查询物流
  ↓
检查配置优先级 (快递100 > 聚合数据 > 模拟)
  ↓
调用真实API
  ↓
格式化返回
  ↓
降级处理（无配置时返回模拟数据）
```
