## 一、问题发现

### 1.1 前后端路由不匹配 (P0)

| 序号 | 前端页面 | 前端调用路径 | 后端定义路径 | 状态 |
|------|----------|--------------|--------------|------|
| 1 | products.html | `/api/user/products` | `/api/products` | ❌ 不匹配 |
| 2 | cart.html | `/api/user/cart` | `/api/cart` | ❌ 不匹配 |

**影响**: 
- 商品列表页无法正常加载
- 购物车页面无法正常加载
- 用户体验严重受损

### 1.2 已实现但未集成的服务 (P1)

| user_behavior_service.py | 用户行为追踪 | ❌ 未集成 |

### 1.3 缺少关键API端点 (P1)

- 商品详情页浏览时缺少行为记录端点
- 购物车页面缺少猜你喜欢推荐区域

## 二、解决方案

### 2.1 路由修复策略

**方案**: 在后端添加 `/api/user/*` 路由别名，保持向后兼容

```python
# 新增别名路由 (与原有路由功能一致)
@app.route('/api/user/products', methods=['GET'])
def get_products_alias():
    return get_products()

@app.route('/api/user/cart', methods=['GET'])
@require_login
def get_cart_alias():
    return get_cart()
```

### 2.2 智能推荐集成

在购物车页面底部添加「猜你喜欢」推荐区域：
- 基于购物车商品类别推荐
- 基于浏览足迹推荐
- 基于购买历史推荐

### 2.3 行为追踪增强

在商品详情页接入 `user_behavior_service.py`：
- 记录商品浏览行为
- 记录停留时长
- 支持关联推荐

## 四、技术实现

### 4.1 路由别名实现

```python
# /api/user/products -> 兼容 /api/products
@app.route('/api/user/products', methods=['GET'])
def get_products_alias():
    return get_products()

# /api/user/cart -> 兼容 /api/cart
@app.route('/api/user/cart', methods=['GET'])
@require_login
def get_cart_alias(user):
    return get_cart(user)
```

### 4.2 购物车推荐区域实现

```javascript
// 页面加载完成后获取推荐
async function loadRecommendations() {
    const userId = getUserIdFromToken(token);
    const resp = await fetch(`/api/user/cart/recommendations?access_token=${token}`);
    const data = await resp.json();
    if (data.success && data.data.length > 0) {
        renderRecommendations(data.data);
    }
}
```

### 4.3 行为追踪实现

```javascript
// 商品详情页加载时记录
async function trackProductView() {
    await fetch('/api/user/behavior/track', {
        method: 'POST',
        headers: {'Content-Type': 'application/json'},
        body: JSON.stringify({
            behavior_type: 'view',
            target_type: 'product',
            target_id: productId,
            duration: pageLoadTime
        })
    });
}
```
