# V10.0 迭代总结

## 迭代背景

本轮迭代从**产品经理深度评审**视角出发，以"客户真实使用场景"为标准，对整个三端系统做全面体检，重点关注：
- 用户交易闭环是否完整
- 运营侧工具是否好用
- 已有接口能力是否被充分呈现
- 前端入口是否存在死链或遗漏
- 统计数据是否准确反映业务现状

---

## 评审发现的问题清单

| 优先级 | 问题描述 | 影响范围 |
|--------|----------|----------|
| **P0** | 用户端首页「我的优惠券」点击 404（`my_coupons.html` 不存在） | 用户端 |
| **P0** | 管理端概览「活跃用户」实际展示的是 `total_applications`（错误字段） | 管理端 |
| **P1** | 用户端订单页仅有列表，无发货状态/退款/确认收货操作 | 用户端 |
| **P1** | 缺少「会员中心」统一入口，积分/签到/优惠券/地址功能孤立 | 用户端 |
| **P1** | 管理端缺少「商品管理」CRUD 页面，商品运营入口缺失 | 管理端 |
| **P1** | 管理端统计看板缺少签到、优惠券使用、库存预警指标 | 管理端 |
| **P1** | 员工端首页菜单缺少「预约管理」和「核销站」入口 | 员工端 |
| **P1** | 员工端订单列表无发货操作、无退款审核能力 | 员工端 |
| **P2** | 数据库缺少商品字段（original_price/sort_order）及购物车表索引 | 数据库 |
| **P2** | 测试框架依赖数据库连接，无法在无 DB 环境下验证结构完整性 | 测试 |

## 本轮完成的迭代内容

### 用户端（business-userH5）

#### 1. 新增 `my_coupons.html` — 修复 P0 级 404
- **功能**：4 Tab（未使用/已使用/已过期/积分兑换）
- **交互**：展示券面金额/折扣、有效期、券码；支持积分兑换优惠券
- **接口**：
  - `GET /api/user/coupons?status=...`
  - `GET /api/coupons/available`
  - `POST /api/user/coupons/exchange`

#### 2. 升级 `orders.html` — 订单生命周期操作页
- **原状态**：仅有简单列表 + 取消按钮
- **升级后**：完整的订单状态机展示与操作
  - 新增 Tab 状态筛选：全部/待支付/待发货/已发货/已完成/退款中/已取消/已退款
  - 每个订单并发请求详情，展示商品明细、物流、收货地址
  - 操作按钮：申请退款（含原因填写弹窗）、确认收货、取消订单
- **接口**：
  - `GET /api/user/orders/<id>` — 订单详情
  - `POST /api/user/orders/<id>/refund` — 申请退款
  - `POST /api/user/orders/<id>/confirm` — 确认收货

#### 3. 新增 `member.html` — 用户资产总入口
- **功能**：
  - 展示昵称/手机/等级/积分/累计积分/余额/连续签到天数
  - 等级进度条（当前积分占下一级所需比例）
  - 快捷入口：我的订单、我的预约、优惠券、收藏夹、每日签到
  - 地址管理入口
  - 积分明细列表（最近 20 条，支持分页）
- **接口**：
  - `GET /api/user/member`
  - `GET /api/user/checkin/status`
  - `GET /api/user/coupons?status=unused`
  - `GET /api/user/addresses`
  - `GET /api/user/points/logs`

#### 4. 更新 `index.html` — 首页菜单优化
- 新增「会员中心」菜单（`member.html`）
- 保留「我的优惠券」菜单（`my_coupons.html`）

---

### 管理端（business-admin）

#### 5. 新增 `products.html` — 商品管理 CRUD 页面
- **功能**：
  - 商品列表（搜索、按门店/分类/状态筛选、分页）
  - 新增/编辑商品弹窗（名称/分类/价格/原价/库存/描述/图片/排序）
  - 上下架切换
  - 删除商品
- **接口**：
  - `GET /api/admin/products`
  - `POST /api/admin/products`
  - `GET /api/admin/products/<id>`
  - `PUT /api/admin/products/<id>`
  - `DELETE /api/admin/products/<id>`

#### 6. 扩展 `app.py` — 管理端后端增强
- 商品列表接口增加 `shop_name` / `original_price` / `description` / `sort_order`
- 新增 `GET /api/admin/products/<id>` 商品详情接口
- 商品更新接口新增允许字段：`original_price` / `sort_order`
- 统计接口 `admin_statistics()` 新增 5 个字段：
  - `today_checkins` — 今日签到人次
  - `month_checkins` — 本月签到人次
  - `used_coupons_today` — 今日优惠券使用量
  - `total_active_coupons` — 当前存量有效券
  - `low_stock_products` — 库存预警商品数（stock<10）

#### 7. 更新 `statistics.html` — 统计看板增强
- 新增第二组指标卡：
  - 今日签到 / 本月签到
  - 今日券使用 / 存量优惠券
  - 注册会员数
  - 库存预警商品数

#### 8. 修复 `index.html` — 概览页数据源纠错
- **原问题**：`activeUsers` 卡片实际显示 `total_applications`（申请单总数），完全错误
- **修复**：改为优先展示 `active_users_7d`（7日活跃），其次 `total_members`（注册会员数）
- 卡片标题改为「注册会员」，趋势描述改为「7日活跃」，语义准确

#### 9. 更新 `index.html` — 新增商品管理菜单
- 侧边栏新增「商品管理」菜单项，含对应 iframe panel

---

### 员工端（business-staffH5）

#### 10. 升级 `index.html` — 首页菜单扩展
- 从 5 个菜单扩展到 7 个
- **新增「📅 预约管理」**：链接到 `bookings.html`
- **新增「✅ 核销站」**：直接打开 `bookings.html?tab=verify`（预约核销页）

#### 11. 升级 `order_list.html` — 员工端订单操作增强
- **原功能**：仅有列表展示，无任何操作
- **升级后**：
  - 新增状态筛选：待发货/已发货/退款中
  - 新增「发货」操作：
    - 弹窗选择物流公司（8家 + 其他）
    - 填写运单号（可选）
    - 调用 `POST /api/staff/orders/<id>/status` 发货
  - 新增「确认完成」操作：已发货订单可标记完成
  - 新增「退款审核」操作：退款中的订单可同意/拒绝
  - 展示物流信息（物流公司 + 运单号）
  - 点击遮罩关闭弹窗

### 数据库（business-common）

#### 12. 新增 `migrate_v10.py` — 数据库迁移脚本
- 幂等工具函数：`safe_alter()` / `safe_create_index()`
- 迁移内容：
  - `business_products.original_price` — 原价字段

  - `business_orders.refund_status` — 退款状态字段
  - 关键索引优化：orders/members/bookings/products
