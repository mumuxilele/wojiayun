## 二、核心功能实现

### 2.1 密码安全升级

#### 问题描述
原系统使用简单SHA256哈希密码，无盐值，易被彩虹表攻击。

#### 解决方案
采用bcrypt算法，支持多版本兼容：

```python
# V35.0 新增功能
- bcrypt哈希（默认rounds=12）
- 密码强度验证（长度/复杂度/弱密码检测）
- 密码版本管理（1=SHA256旧版, 2=bcrypt新版）
- 旧密码兼容验证
```

#### 新增方法
- `_hash_password()` - 支持多版本的密码哈希
- `_verify_password()` - 兼容新旧格式的密码验证
- `_validate_password_strength()` - 密码强度检查
- `reset_password()` - 用户自助密码重置
- `force_reset_password()` - 管理员强制重置

### 2.2 账户安全增强

#### 问题描述
无登录失败锁定机制，容易被暴力破解。

#### 解决方案
实现账户锁定机制：

```python
# V35.0 新增配置
LOGIN_MAX_FAILURES = 5   # 最大失败次数
LOGIN_LOCK_DURATION = 30 # 锁定30分钟

# 新增字段
- login_fail_count: 连续登录失败次数
- lock_until: 账户锁定截止时间
```

#### 新增方法
- `_check_account_locked()` - 检查账户是否被锁定
- `_handle_login_failure()` - 处理登录失败
- `_reset_login_failures()` - 重置失败计数

### 2.3 Token安全增强

#### 问题描述
原Token生成方式简单，无签名验证，分布式部署不兼容。

#### 解决方案
采用JWT + Redis方案：

```python
# V35.0 Token特性
- JWT格式签名
- 2小时Access Token + 7天Refresh Token
- Redis存储Token黑名单
- 支持主动失效
```

#### 新增方法
- `_generate_token()` - 生成JWT Token
- `_verify_token()` - 验证JWT Token
- `_revoke_token()` - 主动失效Token

### 2.4 库存并发优化

#### 问题描述
订单创建时库存扣减无锁保护，高并发可能超卖。

#### 解决方案
使用悲观锁（SELECT ... FOR UPDATE）：

```python
# V35.0 新增方法
- _lock_stock_with_pessimistic_lock(): 悲观锁扣减库存
- _lock_stock_with_retry(): 带重试的库存扣减
- _unlock_stock(): 库存回滚
```

#### 技术特性
- FOR UPDATE行锁
- 事务自动管理
- 重试机制（指数退避）
- 失败自动回滚

### 2.5 Redis缓存服务

#### 新增文件
`business-common/redis_cache_v35.py`

#### 核心功能

class RedisCache:

## 八、性能影响

### 8.1 密码验证性能

| 方案 | 验证耗时 | 说明 |
|------|---------|------|
| SHA256 | ~0.01ms | 最快 |
| bcrypt(rounds=10) | ~1ms | 适中 |
| bcrypt(rounds=12) | ~3ms | 当前使用 |
| bcrypt(rounds=14) | ~12ms | 更安全但慢 |

V35.0使用rounds=12，在安全性和性能间取得平衡。

### 8.2 Token验证性能

JWT验证约0.1-0.5ms，Redis检查约0.5-1ms。

### 8.3 库存扣减性能

悲观锁增加约1-3ms锁定时间，但确保并发安全。
