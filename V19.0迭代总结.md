## 二、核心变更

### P0 - 安全加固

#### 1. 管理端认证修复
- **问题**: `business-admin/app.py` 中8个写操作路由缺少 `@require_admin` 装饰器，仅依赖手动 `get_current_admin()` 检查，存在认证绕过风险
- **修复**: 为以下路由补充装饰器：
  - `admin_update_shop` - 门店更新
  - `admin_delete_shop` - 门店删除
  - `update_venue` - 场地更新
  - `delete_venue` - 场地删除
  - `update_venue_booking` - 预约更新

#### 2. 秒杀防超卖
- **问题**: 促销秒杀活动在高并发场景下可能出现超卖
- **修复**: 在库存扣减逻辑中添加 `FOR UPDATE` 行级锁，确保并发安全

### P1 - 功能增强

#### 3. 用户自助退款申请
- **新增接口**: `POST /api/user/orders/<order_id>/refund`
- **功能特性**:
  - 仅允许已支付/已发货订单申请退款
  - 必填退款原因
  - 防止重复申请（检查现有退款状态）
  - 自动发送站内通知给员工端
  - WebSocket实时推送给员工
- **前端支持**: `orders.html` 已有退款弹窗和提交逻辑

#### 4. 申请超时预警机制
- **新增服务**: `business-common/application_monitor.py`
- **核心功能**:
  - 定时检测挂起超过24小时的申请单
  - 自动发送站内通知和WebSocket推送
  - 4小时冷却期避免重复提醒
  - 按超时时长分级统计（24-48h/48-72h/72h+）
- **新增接口**:
  - `GET /api/staff/applications/overdue` - 超时统计
  - `GET /api/staff/applications/<app_id>/timeline` - 申请时间线

### P2 - 体验优化

#### 5. 管理端概览页ECharts升级
- **新增图表**:
  - 近7天订单趋势图（折线+柱状混合）
  - 申请单状态分布饼图
  - 收入来源分析横向柱状图
- **新增面板**: 超时申请预警面板（展示最紧急的5条）
- **技术实现**: 引入ECharts 5.4.3，响应式布局

#### 6. 商品多维度排序
- **已有功能验证**: 商品列表已支持多种排序方式
  - `price_asc` / `price_desc` - 价格升序/降序
  - `sales` - 销量排序
  - `hot` - 热度排序
  - `new` - 新品排序
  - `default` - 综合排序

## 五、数据库变更

### 新增字段
```sql
-- business_orders 表
ALTER TABLE business_orders 
ADD COLUMN refund_status VARCHAR(20) DEFAULT NULL COMMENT '退款状态',
ADD COLUMN refund_requested_at DATETIME DEFAULT NULL COMMENT '退款申请时间';
```

### 新增索引
```sql
-- 申请单状态+时间索引（用于超时查询）
CREATE INDEX idx_apps_status_created ON business_applications(status, created_at);

-- 订单退款状态索引
CREATE INDEX idx_orders_refund_status ON business_orders(refund_status, order_status);
```
