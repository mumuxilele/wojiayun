#### 新增功能

| 功能 | 说明 | 优先级 |

| 智能物流识别 | auto_detect_company | P1 |

| 状态变更通知 | _notify_status_change | P2 |
| 时效分析 | analyze_delivery_performance | P2 |

#### 物流公司智能识别

支持自动识别的快递公司：

- 顺丰速运 (SF) - 12/15位数字或SF开头
- 京东物流 (JD) - JD开头或16位数字
- 邮政EMS - EE/RA开头
- 圆通速递 (YTO) - YT开头或12位数字
- 中通快递 (ZTO) - 7000开头
- 申通快递 (STO) - 88/68开头
- 韵达快递 (YD) - 10-15位数字
- 极兔速递 (JTSD) - JT开头

### 2.3 后端分层架构 (service_base.py / repository_base.py)

#### 服务层基类

```python
# 基础服务
BaseService
  ├── health_check()
  ├── uptime
  └── now() / now_str()

# CRUD服务
CRUDService
  ├── create(data)
  ├── get_by_id(id)
  ├── get_all(filters, order_by, limit)
  ├── update(id, data)
  ├── delete(id)
  └── count(filters)

# 事务服务
TransactionService
  └── execute_transaction(operations)
```

#### 仓储层基类

```python
# 基础仓储
BaseRepository
  ├── find_by_id(id)
  ├── find_all(limit, offset)
  ├── find_by_field(field, value)
  ├── insert(data)
  ├── update(id, data)
  ├── delete(id)
  ├── count(where, params)
  └── transaction() [context manager]

# 领域仓储
MemberRepository
  ├── find_by_phone(phone)
  ├── find_by_openid(openid)
  ├── update_points(user_id, points_delta)
  └── update_balance(user_id, amount)

OrderRepository
  ├── find_by_order_no(order_no)
  ├── find_by_user(user_id, status, limit)
  └── update_status(order_id, status)

PaymentRepository
  ├── find_by_pay_no(pay_no)
  ├── find_pending(limit)
  └── find_by_user(user_id, limit)

ProductRepository
  ├── find_active(category_id, limit)
  ├── increment_view(product_id)
  └── increment_sales(product_id, quantity)
```
