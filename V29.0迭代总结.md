# V29.0 迭代总结

**版本**: V29.0  
**日期**: 2026-04-10  
**迭代主题**: 申请审批系统全面升级

---

## 一、迭代背景

为满足物业管理和租户服务的业务需求，本次迭代实现了完整的申请审批系统，支持8种业务申请类型，覆盖物业申请、设备申请和活动申请等多个场景。

---

## 二、核心功能清单

### 2.1 业务申请类型 (8种)

| 申请代码 | 申请名称 | 优先级 | 分类 | 审批流程 | 特殊功能 |
|---------|---------|-------|------|---------|---------|
| overtime | 加班申请 | P0 | 物业申请 | 一级审批 | 区域/时间/人数 |
| chair_placement | 候餐椅临时摆放申请 | P1 | 物业申请 | 一级审批 | 到期自动提醒 |
| key_return | 备用钥匙回收申请 | P1 | 物业申请 | 一级审批 | 归档钥匙台账 |
| tenant_activity | 租户活动申请 | P0 | 活动申请 | 二级审批 | 自动生成工单 |
| monitor_query | 监控录像调用申请 | P0 | 物业申请 | 一级审批 | 全程审计留痕 |
| hvac_overtime | 加时鲜风供应申请 | P0 | 设备申请 | 工程部审批 | 联动暖通系统 |
| cargo_lift | 货梯使用预约申请 | P0 | 设备申请 | 一级审批 | 时间冲突检测 |
| signage_print | 指示牌印制申请 | P1 | 物业申请 | 一级审批 | 设计稿上传 |

### 2.2 通用能力

- **历史记录查询**: 支持按类型、状态筛选
- **一键复用**: 常用申请快速复制提交
- **到期提醒**: 候餐椅、货梯预约到期自动提醒
- **附件上传**: 支持图片、PDF、设计稿等
- **企微推送**: 审批结果推送


### 6.2 多级审批流程

```python
# 审批流程配置
approve_flow = {
    'steps': 2,
    'nodes': [
        {'step': 1, 'name': '物业初审', 'role': 'property_manager'},
        {'step': 2, 'name': '运营终审', 'role': 'operation_manager'}
    ]
}
```

### 6.3 货梯时间冲突检测

```python
# 查询同一货梯同一时间段的已批准申请
SELECT * FROM business_applications 
WHERE app_type='cargo_lift' 
  AND status IN ('pending', 'approved')
  AND JSON_EXTRACT(form_data, '$.lift_no') = ?
  AND JSON_EXTRACT(form_data, '$.use_date') = ?
  AND time_range_overlap(start_time, end_time)
```

### 6.4 到期提醒机制

```python
# 自动设置提醒时间
if app_type.need_remind:
    end_datetime = parse(form_data.end_date + form_data.end_time)
    remind_time = end_datetime - timedelta(hours=remind_before_hours)
    create_remind_record(app_id, remind_time)
```
