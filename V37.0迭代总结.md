#### 后端逻辑
```python
# 积分抵扣比例
POINTS_RATE = 100  # 100积分 = 1元

# 订单创建时处理积分抵扣
use_points = data.get('use_points', 0)
if use_points > 0:
    # 校验用户积分余额
    member = db.get_one("SELECT points FROM business_members WHERE user_id=%s", [uid])
    user_points = member.get('points', 0)
    
    # 计算最大可用积分
    max_points = min(user_points, POINTS_RATE * actual_amount)
    use_points = min(use_points, max_points)
    
    # 扣除积分
    points_deduction = use_points / POINTS_RATE
    actual_amount -= points_deduction
    
    # 记录积分日志
    add_points_log(user_id, -use_points, 'order_deduction', ...)
```

# 根据会员等级获取折扣率
level_info = db.get_one("""
    SELECT level_name, discount_rate 
    FROM business_member_levels 
    WHERE level_code=%s
""", [member_level])

discount_rate = float(level_info['discount_rate'])
member_price = round(regular_price * discount_rate, 2)
member_savings = regular_price - member_price

## 四、用户体验闭环

### 会员价感知闭环
```
商品详情页 → 会员价展示 → 会员升级动力 → 消费转化
```

### 积分消费闭环
```
积攒积分（签到/消费/评价）→ 查看积分余额 → 结算页抵扣 → 积分减少 → 继续积攒
```
