## 四、V24.0 迭代计划

### 4.1 迭代主题
**核心**: 支付闭环 + 架构升级 + 营销增强

### 4.2 功能清单

#### F1: 微信支付V3正式集成 (P0)
```
需求:
- 对接微信支付V3 API (Native支付)
- 实现支付回调AES-GCM解密验签
- 实现支付单查询接口
- 支持H5调起支付

技术:
- 安装 wechatpayv3 库
- 配置商户号、序列号、APIv3密钥、私钥
- 实现回调验签 + 订单状态更新

文件:
- business-common/payment_service.py (增强)
- business-userH5/app.py (新增回调端点)
```

#### F2: 支付宝支付正式集成 (P0)
```
需求:
- 对接支付宝统一收单支付
- 实现RSA2签名验签
- 支持手机网站支付(wap)和电脑网站支付(page)

技术:
- 安装 alipay-sdk-python
- 配置应用ID、私钥、支付宝公钥

文件:
- business-common/payment_service.py (增强)
- business-userH5/app.py (新增回调端点)
```

#### F3: 物流追踪API集成 (P0)
```
需求:
- 对接快递100/菜鸟物流追踪API
- 订单详情展示实时物流
- 物流状态变更自动通知

技术:
- 快递100付费API (或聚合数据)
- 物流状态变更触发WebSocket推送

文件:
- business-common/logistics_service.py (增强)
- business-userH5/app.py (新增查询端点)
- business-userH5/order_detail.html (物流展示)
```

#### F4: 代码架构重构 (P1)
```
目标: 分层架构 (Controller/Service/Repository)

Phase 1: 抽取公共模块
- 认证服务 (auth_service.py)
- 支付服务 (payment_service.py) - 已存在，增强
- 通知服务 (notification_service.py)

Phase 2: 业务服务化
- 订单服务 (order_service.py)
- 会员服务 (member_service.py)
- 商品服务 (product_service.py)

Phase 3: 数据访问层
- 统一Repository基类
- 数据访问抽象

文件:
- business-common/services/ (新建)
- business-common/repositories/ (新建)
```

#### F5: 拼团活动系统 (P1)
```
需求:
- 拼团活动创建 (2人团/5人团/自定义)
- 拼团订单管理
- 团长/团员机制
- 拼团成功/失败处理

技术:
- 新表: business_group_activities
- 新表: business_group_orders

API:
- POST /api/user/group-buy/join - 参与拼团
- GET /api/user/group-buy/my - 我的拼团
- POST /api/staff/group-activities - 管理拼团
```

#### F6: RBAC细粒度权限 (P1)
```
需求:
- 角色管理 (超级管理员/运营/财务/客服)
- 权限点配置 (功能权限 + 数据权限)
- 菜单权限 + API权限
- 操作审计

技术:
- 新表: business_roles
- 新表: business_permissions
- 新表: business_role_permissions
- 新表: business_user_roles
```

#### F7: API版本管理 (P1)
```
需求:
- 支持 /api/v1/ 和 /api/v2/ 共存
- 平滑升级，过渡期6个月

实现:
- Blueprints版本分组
- 兼容处理层
```

#### F8: 运营数据看板 (P1)
```
需求:
- 用户画像 (消费能力/偏好)
- 转化漏斗 (浏览→下单→支付)
- 留存分析 (次日/7日/30日)

技术:
- 埋点事件收集
- 数据仓库建模
- ECharts可视化
```

#### F9: WebSocket实时推送 (P2)
```
需求:
- 订单状态变更推送
- 物流状态变更推送
- 站内消息实时通知

实现:
- 优化 websocket_service.py
- 前端WebSocket连接集成
```

#### F10: 短信/邮件通知 (P2)
```
需求:
- 用户注册验证码
- 订单状态通知
- 物流通知

技术:
- 阿里云短信服务
- SMTP邮件服务
```
