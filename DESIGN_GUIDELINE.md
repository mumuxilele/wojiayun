# 物业租户双向H5页面UI&交互设计规范（统一主题色·内嵌版）

> **版本**: v1.0  
> **适用**: 用户端（租户）+ 员工端（物业）H5页面  
> **生效日期**: 2026-04-11  
> **规范性质**: **强制性** - 所有新开发或旧页面重构必须遵循

---

## 一、规范总则

### 1.1 适用范围
- 本规范适用于**嵌入APP/小程序的H5页面**
- **不设计独立顶部标题栏、底部导航栏**，导航与标题由宿主容器提供，仅设计页面内容区域

### 1.2 主题色统一
- 员工端、用户端品牌主题色统一为 **#30c2b3**
- 两端组件样式、交互逻辑、日期控件完全统一，视觉风格一致

### 1.3 设计基准
- 设计稿基准宽度：**375px**
- 适配移动端各类屏幕
- 遵循极简、高效、质感设计原则
- 满足租户便捷申请、物业高效审批核心需求

### 1.4 覆盖范围
- 覆盖全部9项业务申请 + 1项通用能力
- 所有表单组件标准化，可直接落地设计与开发

## 三、通用组件设计规范（两端完全统一）

### 3.1 单行文本输入框

**样式规范：**
- 高度：**48px**
- 边框：无边框，仅底部 **1px 分割线（#E5E6EB）**
- 背景：透明
- 内边距：左右 0，垂直居中

**交互状态：**
- 聚焦：底部分割线切换为 **#30c2b3**
- 错误：底部分割线 **#F53F3F**，下方显示红色错误提示（14px）
- 禁用：文字 **#C9CDD4**，分割线 **#C9CDD4**

**适用场景：**
- 姓名、人数、数量、编号、联系电话等短文本填写

**代码示例：**
```css
.form-input {
    width: 100%;
    height: 48px;
    padding: 12px 0;
    border: none;
    border-bottom: 1px solid #E5E6EB;
    font-size: 16px;
    color: #1D2129;
    background: transparent;
    outline: none;
}
.form-input:focus {
    border-bottom-color: #30c2b3;
}
.form-input.error {
    border-bottom-color: #F53F3F;
}
```

### 3.2 多行文本输入框

**样式规范：**
- 最小高度：**80px**
- 最大高度：**160px**
- 圆角：**4px**
- 边框：无边框，背景 **#FFFFFF**
- 内边距：**12px**

**交互状态：**
- 聚焦：**1px #30c2b3 描边**（box-shadow 实现）
- 右下角显示字数统计（已输入/最大200字，12px #86909C）

**适用场景：**
- 申请原因、活动内容、布置要求、审批意见等长文本

**代码示例：**
```css
.form-textarea {
    width: 100%;
    min-height: 80px;
    max-height: 160px;
    padding: 12px;
    border: none;
    border-radius: 4px;
    font-size: 16px;
    color: #1D2129;
    background: #FFFFFF;
    resize: vertical;
    outline: none;
}
.form-textarea:focus {
    box-shadow: 0 0 0 1px #30c2b3;
}
```

### 3.3 下拉选择框

**样式规范：**
- 高度：**48px**
- 布局：左侧标签 + 右侧选中内容 + **24px 下拉箭头（#86909C）**
- 边框：底部 **1px #E5E6EB**
- 整体可点击

**弹窗规范：**
- 底部弹出半屏选择面板
- 选项高度：**44px**
- 面板最大占屏 **60%**
- 顶部标题 + 关闭按钮

**交互状态：**
- 选项≥5项时显示顶部搜索框
- 选中：选项背景 **#E6FAF7**，文字 **#30c2b3**
- 禁用：文字、箭头置灰

**适用场景：**
- 区域、货梯、权限范围、申请类型等固定选项选择

### 3.4 日期/时间控件（全局唯一样式）

**显示格式：**
- 单日期：`YYYY-MM-DD`
- 日期+时间：`YYYY-MM-DD HH:MM`
- 时间段：`YYYY-MM-DD HH:MM - YYYY-MM-DD HH:MM`

**样式规范：**
- 输入框高度：**48px**
- 样式同单行输入框

**交互规范：**
- 点击底部弹出日历/时间滚轮
- 时间间隔：**15分钟**
- 今日高亮：圆形描边 **#30c2b3**
- 选中日期：圆形填充 **#30c2b3**，文字白色
- 自动校验：开始时间＜结束时间
- 已占用/过期时间置灰，不可选择

**统一规则：**
- 全项目时间格式统一，无多余符号

### 3.5 数字选择器

**样式规范：**
- 整体宽度：**140px**
- 组成：减少按钮 + 输入框 + 增加按钮
- 按钮尺寸：**32px×32px**
- 按钮圆角：**4px**
- 按钮边框：**1px #E5E6EB**

**交互规范：**
- 输入框数字居中
- 最小值默认为 **1**
- 支持手动输入/按钮增减
- 超出数值范围触发红色提示，禁止提交

**适用场景：**
- 加班人数、椅子数量、指示牌数量等

### 3.6 文件上传组件

**样式规范：**
- 圆角：**4px**
- 边框：**1px dashed #E5E6EB**
- 高度：**80px**
- 内置上传图标 + 提示文字

**交互规范：**
- 支持拍照/相册选择
- 格式仅限：**PNG/JPG**
- 单文件最大：**10M**
- 上传后显示缩略图，支持删除、重新上传
- 最多支持上传 **9张**

**适用场景：**
- 租户活动效果图、指示牌设计稿、报事附件上传

## 四、用户端（租户）H5专属规范

### 4.1 页面布局
- 单列流式布局，页面顶部居左显示页面标题，无多余导航
- 表单统一收纳在白色卡片内，依次垂直排列
- 提交按钮固定在页面底部，不随页面滚动
- 申请列表采用卡片式布局，按时间倒序排列

### 4.2 核心交互
1. 表单必填项标注红色 `*`，非必填项灰色提示
2. 常用申请支持**一键复用**，自动填充历史信息，可修改后提交
3. 审批结果同步推送，申请状态实时更新
4. 餐饮候餐椅申请到期前30分钟，弹出全局提醒
5. 未提交表单自动临时保存，重新进入可继续编辑

### 4.3 视觉要求
- 以留白、简洁卡片、柔和阴影打造质感，无复杂视觉效果
- 图标统一线性风格，无夸张色彩
- 文字层级少，重点突出操作按钮
- 加载、反馈动画柔和，无过度动效

## 五、员工端（物业）H5专属规范

### 5.1 页面布局
- 模块化布局，审批信息按业务分组展示
- 多级审批页面顶部添加审批步骤条，流程可视化
- 审批意见、操作按钮置于页面底部，便捷操作
- 台账、审计日志采用列表+筛选栏布局，数据清晰

### 5.2 核心交互
1. 表单提交前自动逻辑校验，不符合要求禁止提交
2. 货梯申请时间冲突实时红色提示，支持重新选择
3. 审批通过后自动触发系统联动（门禁、暖通、梯控）
4. 监控调用申请全程操作留痕，记录不可修改
5. 备用钥匙回收确认后，自动归档至管理台账
6. 租户活动审批通过，自动生成现场配合工单

### 5.3 视觉要求
- 严格遵循品牌主题色，操作按钮、核心图标均使用 **#30c2b3**
- 信息层级分明，审批状态、关键业务数据高亮显示
- 组件排列紧凑但不拥挤，操作按钮按业务优先级排列

## 六、业务页面通用规范

### 6.1 页面结构标准

**申请表单页面：**
```
页面标题
├── 信息卡片（申请类型图标+名称）
│   └── 申请标题输入框
├── 表单卡片（动态表单字段）
├── 附件卡片（如需）
├── 备注卡片
└── 底部按钮栏（存草稿 + 提交）
```

**审批页面：**
```
页面标题
├── 统计卡片（待审批/今日/累计）
├── 筛选栏
├── 申请列表卡片
└── 审批弹窗（通过/拒绝）
```

### 6.2 通用能力规范

**FR-BP-060 通用能力：**
1. **历史记录查询**：所有申请支持按状态、时间筛选，展示完整申请信息与审批流程
2. **重复提交**：申请详情页底部设置「再次申请」按钮，一键复用历史表单数据
3. **消息推送**：审批结果同步至企微，H5页面同步更新申请状态
4. **数据缓存**：未提交表单自动临时保存，重新进入可继续编辑

### 6.3 加载与反馈

**加载动画：**
- 页面加载、提交、审批均显示轻量加载动画
- 加载提示文字：**14px #86909C**

**操作反馈：**
- 操作成功/失败均弹出轻量提示（Toast）
- 显示时长：**2秒自动消失**
- 成功：绿色提示
- 失败：红色提示

**权限控制：**
- 员工端按角色显示对应审批/管理权限
- 无权限功能隐藏

## 八、设计验收标准

1. **视觉一致性**：两端色彩、组件、文字完全符合本规范，无视觉偏差
2. **交互统一性**：所有组件交互逻辑统一，操作流程无卡顿，反馈及时
3. **业务完整性**：所有申请表单、审批流程、系统联动、自动提醒功能正常
4. **体验适配性**：两端操作无冗余，适配所有屏幕尺寸，符合移动端操作习惯
