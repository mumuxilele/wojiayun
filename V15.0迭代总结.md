## 一、迭代概览

V15.0 从产品经理视角出发，对系统进行了全面的安全审计和功能完善。本次迭代以"安全第一、体验升级"为核心思路，涵盖了 3 个 P0 安全修复、5 个功能增强、13 个前端页面的 XSS 防护加固，以及全新的自动化测试框架升级。

## 二、变更明细

### 2.1 安全修复（P0/P1）

| ID | 优先级 | 改动项 | 涉及文件 |
|----|--------|--------|---------|
| V15-1 | P0 | Token明文缓存修复 - 使用SHA256哈希替代明文Token作为缓存key | `business-common/auth.py` |
| V15-2 | P0 | 管理端认证绕过修复 - before_request钩子白名单最小化 | `business-admin/app.py` |
| V15-3 | P1 | 退款金额验证 - admin/staff端退款金额不能超过实付金额 | `business-admin/app.py`, `business-staffH5/app.py` |

**V15-1 详细说明**:
- 新增 `_hash_key()` 函数，使用 `hashlib.sha256` 对缓存key哈希
- `_token_cache` 字典中不再存储明文Token，仅存储SHA256哈希值
- `invalidate_token()` 函数同步使用哈希key进行缓存失效
- 缓存TTL保持60秒不变

**V15-2 详细说明**:
- `@app.before_request` + `check_admin()` 钩子覆盖所有管理端路由
- 豁免路径最小化为 `/health`、`/debugtest`、`/api/admin/test`
- 消除了之前部分API路由无需认证即可访问的安全隐患

**V15-3 详细说明**:
- 管理端退款支持部分退款金额（`refund_amount` 参数）
- 校验退款金额不能为负数、不能超过实付金额（允许0.01元浮点误差）
- 员工端退款同步添加金额校验逻辑

## 三、技术架构改进

### 3.1 测试框架升级
- **测试模式变更**: 从模块导入测试改为源码分析测试
- **优势**: 无需 pymysql、flask 等运行时依赖，可在任何环境快速运行
- **方法**: 通过正则解析源码中的路由定义、函数签名、字符串常量进行验证

### 3.2 安全基础设施
- **Token缓存**: SHA256哈希key + TTL 60s，避免明文Token内存泄露
- **认证钩子**: before_request全局拦截 + 最小白名单
- **XSS防护**: 统一escapeHtml函数 + 13个页面加固
- **金额校验**: 服务端校验退款金额上限，防止超额退款
